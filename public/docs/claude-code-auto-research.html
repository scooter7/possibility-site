1. Overview: The Agentic Loop

Claude Code is built on an agentic loop: Claude autonomously reads files, runs commands, searches code, edits files, and reasons through problems without requiring approval for every step. The system automatically manages tool execution, permissions, context, and can delegate work to specialized subagents running in parallel.

The core autonomy comes from several interlocking systems:

  • Built-in tools for file operations and code execution
  • Subagents for specialized task delegation
  • Agent teams for coordinated parallel work
  • Hooks for deterministic automation
  • Memory systems for persistent learning
  • Scheduled tasks for recurring automation

2. The Explore Agent and Auto-Research

How It Works

Claude Code includes a built-in Explore subagent optimized specifically for research and codebase analysis. It runs on the Haiku model for fast, low-latency responses and has read-only access (Glob, Read, Grep, Bash). It cannot modify files during exploration.

Exploration results stay out of your main conversation context (separate context window), keeping your primary session lean.

Thoroughness Levels

Quick

Targeted lookups. "Find the auth config file."

Medium

Balanced exploration. "How does the API handle rate limiting?"

Very Thorough

Comprehensive analysis across multiple locations and naming conventions. "Map the entire authentication flow from login to token refresh."

Built-in Research Tools

ToolPurposeAuto-Approved
GlobFind files by patternYes
GrepSearch file contents with regexYes
ReadRead any file in working directoryYes
WebSearchSearch the web for current informationYes
WebFetchFetch and parse web page contentYes
BashRun terminal commandsRequires permission

Parallel Research Pattern

# Single-agent research
claude -p "Find all database migrations and summarize their purpose"

# Multi-agent parallel research (in conversation)
"Research the authentication, database, and API modules in parallel using separate subagents"

Cost tip: Running multiple subagents that each return detailed results can consume significant context. Request summaries to limit what returns to the main conversation.

# Example: .claude/agents/security-auditor.md
---
name: security-auditor
description: Scans codebase for OWASP Top 10 vulnerabilities
tools: Read, Glob, Grep, Bash

---

You are a senior security engineer. When invoked:
1. Search for common vulnerability patterns

Foreground vs Background

  • Foreground: Blocks main conversation, handles permission prompts interactively
  • Background: Runs concurrently (press Ctrl+B to move to background). Pre-approves permissions before launching so the agent doesn't get blocked mid-run.

Subagent Memory

Subagents can maintain persistent memory across sessions:

---
name: code-reviewer
memory: user    # Options: user, project, local
---
  • user: ~/.claude/agent-memory/<agent-name>/ (across all projects)
  • project: .claude/agent-memory/<agent-name>/ (project-specific, shared via git)
  • local: .claude/agent-memory-local/<agent-name>/ (project-specific, not versioned)

4. Agent Teams: Coordinated Parallel Work

Experimental: Agent teams are disabled by default. Enable with CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 in settings or environment.

What Are Agent Teams?

Multiple Claude Code instances working together with a lead agent coordinating shared tasks, messaging, and results. Unlike subagents (which report back only to the main agent), agent team members communicate directly with each other via a mailbox system.

Best Use Cases

Parallel code review

Security reviewer, performance reviewer, and test coverage reviewer run simultaneously.

Competing hypotheses

Multiple agents test different root-cause theories for a bug.

New feature modules

Teammates each own separate pieces without file conflicts.

Cross-layer coordination

Frontend, backend, and test changes handled in parallel.

Display Modes

  • In-process (default): All teammates run in main terminal. Use Shift+Down to cycle through teammates.
  • Split panes (requires tmux or iTerm2): Each teammate gets its own pane. See all output simultaneously.

Controlling Teams

# Natural language team creation
"Create an agent team to explore this from different angles:
 one on UX, one on architecture, one playing devil's advocate."

# Team lead commands
"Spawn 4 teammates to refactor these modules in parallel"
"Assign the auth module to teammate-1"
"Ask researcher teammate to shut down"
"Clean up the team"

Limitations

  • No in-process session resumption (/resume and /rewind don't restore teammates)
  • One team per session, no nested teams
  • The session that creates the team is lead for life
  • Split panes require tmux or iTerm2 (not VS Code integrated terminal)

5. Auto Mode (Research Preview)

Announced by Anthropic as a research preview rolling out March 12, 2026. Instead of prompting the developer for every file write or shell command, Claude itself classifies risk and decides which actions to auto-approve:

  • Read-only operations on project files: auto-approved
  • Shell commands with broad filesystem or network access: more likely to surface a prompt
  • Ships with prompt injection safeguards (acknowledged as incomplete)
  • Organizations can disable auto mode entirely via MDM or file-based OS policies

Separate from headless mode: Headless mode (-p or --print flag) has existed longer for non-interactive use in CI/CD, pre-commit hooks, and build scripts.

6. Hooks: Deterministic Automation

Hooks are scripts that run automatically at specific lifecycle points in Claude Code. Unlike skills (which Claude applies at its judgment), hooks always execute when their event fires.

Hook Events

EventWhen It FiresUse Case
SessionStartSession begins or resumesInject context, setup environment
PreToolUseBefore tool executes (can block)Validate commands, enforce rules
PostToolUseAfter tool succeedsAuto-format, run linters
UserPromptSubmitWhen you submit a promptPre-process input
StopClaude finishes respondingCheck completeness
SubagentStartSubagent spawnedLog or setup
SubagentStopSubagent finishesCleanup, process results
PreCompactBefore context compactionPreserve critical context

Hook Types

  • Command hooks: Run shell scripts
  • Prompt hooks: Single-turn LLM evaluation
  • Agent hooks: Multi-turn verification with tools
  • HTTP hooks: POST to external services

Examples

// Auto-format after edits (in settings.json)
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write"
      }]
    }]
  }
}

// Block protected files
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "./.claude/hooks/protect-files.sh"
      }]
    }]
  }
}

7. Memory Systems: Persistent Learning

CLAUDE.md Files

Persistent instructions that load every session.

LocationScope
~/.claude/CLAUDE.mdAll projects (user-level)
./CLAUDE.mdSingle project (team-shared)
Managed policyOrganization-wide (admin-controlled)

Best practice: keep under 200 lines per file. Include build commands, code style, and workflow rules. Use @file references to import additional context.

Auto Memory

Notes Claude writes itself based on discoveries and your corrections. Stored per-project at ~/.claude/projects/<project>/memory/. Machine-local, does not sync across machines.

  • Claude reads the first 200 lines of MEMORY.md at session start
  • Learns from corrections and preferences
  • Saves insights to topic files (e.g., debugging.md, api-conventions.md)
  • Toggle with /memory command

8. Scheduled Tasks and Recurring Automation

The /loop Command

/loop 5m check if the deployment finished
/loop check the build every 2 hours
/loop 30m /review-pr 1234
/loop 24h audit the codebase for security issues

Supported units: s (seconds), m (minutes), h (hours), d (days). Default interval: 10 minutes.

One-Time Reminders

remind me at 3pm to push the release branch
in 45 minutes, check whether integration tests passed

Timing Details

  • Tasks fire between your turns (not during Claude's response)
  • All times in local timezone
  • Recurring tasks auto-expire 3 days after creation
  • Up to 50 scheduled tasks per session

9. MCP: External Tool Integration

The Model Context Protocol (MCP) connects Claude to external tools and services without custom tool execution code.

Setup

# Quick interactive setup
claude mcp add

# Manual configuration (.mcp.json)
{
  "mcpServers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"]
    },
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}

10. Agent SDK: Programmatic Agents

The Claude Agent SDK (formerly "Claude Code SDK") lets you build production AI agents with Claude as a library. Available in Python and TypeScript.

Installation

# Python
pip install claude-agent-sdk

# TypeScript
npm install @anthropic-ai/claude-agent-sdk

Basic Usage (Python)

import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions

async def main():
    async for message in query(
        prompt="Find and fix the bug in auth.py",
        options=ClaudeAgentOptions(
            allowed_tools=["Read", "Edit", "Bash"]
        ),
    ):
        if hasattr(message, "result"):
            print(message.result)

asyncio.run(main())

Key Features

  • Same tools, agent loop, and context management that power Claude Code
  • Custom tools defined as Python functions (in-process MCP servers)
  • Hooks for programmatic agent behavior control
  • Subagent spawning and coordination
  • Session persistence and resumption
  • MCP server integration

Deployment Providers

  • Claude API (direct)
  • Amazon Bedrock
  • Google Vertex AI
  • Microsoft Azure Foundry

12. Advanced Autonomous Patterns

Research to Implementation Pipeline

# Session 1: Research (Explore agent)
"Use the Explore agent to research the current authentication system in detail"

# Session 2: Planning (Plan Mode)
claude --permission-mode plan
"Based on the research, create an implementation plan for OAuth migration"

# Session 3: Implementation (Normal Mode)
"Implement the plan from session 2 with verification"

Parallel Hypothesis Testing

"Create an agent team with 5 investigators:
 - One researching database index strategy
 - One investigating caching layer
 - One testing query optimization
 - One analyzing slow query logs
 - One researching alternative database engines

Have them communicate findings and converge on a solution"

Scheduled Monitoring

/loop 1h check deployment health and report any issues
/loop 30m run security audit and flag vulnerabilities
/loop 6h update project status report with latest metrics

CI/CD Integration

#!/bin/bash
# Run Claude Code in CI/CD for code analysis
claude -p "Analyze PR #$PR_NUMBER for quality, security, and coverage" \
  --output-format json > analysis.json

gh pr comment $PR_NUMBER -b "$(cat analysis.json)"
